⏱️ Node.js timers Module Cheat Sheet

setTimeout, setInterval, setImmediate & advanced control — Node.js 18+

💡 Golden Rule: Timers are global — no import needed. Use setTimeout for delays, setInterval for recurring tasks, setImmediate for the next event loop iteration. Node.js timers are not guaranteed to be precise — they're minimum delays.
⏰ setTimeout & clearTimeout
Timeout

setTimeout / clearTimeout

setTimeout(callback, delay?, ...args) / clearTimeout(timeout)

Schedule a callback after a minimum delay (in milliseconds). Returns a Timeout object.

// Basic timeout
const timeout = setTimeout(() => {
  console.log('Executed after 1 second');
}, 1000);

// With arguments
setTimeout((name, age) => {
  console.log(`Hello ${name}, age ${age}`);
}, 500, 'Alice', 30);

// Clear before execution
const timeout2 = setTimeout(() => {
  console.log('Will never run');
}, 1000);
clearTimeout(timeout2);

// Zero delay (schedules on next tick)
setTimeout(() => {
  console.log('Runs after I/O events');
}, 0);
⚠️ setTimeout(fn, 0) schedules on the next event loop tick, but after I/O events. Not guaranteed to run immediately.
Advanced

timeout.ref() / timeout.unref() / hasRef()

timeout.ref() / timeout.unref() / timeout.hasRef()

Control whether the timer keeps the event loop alive. unref() allows the program to exit if this is the only active timer.

const timeout = setTimeout(() => {
  console.log('Will run, but won't keep the process alive');
}, 5000);

// Unref: won't prevent process exit
timeout.unref();
console.log(timeout.hasRef()); // false

// Ref: keep the event loop alive (default)
timeout.ref();

// ⚠️ Careful: if process exits, unref'd timer may be garbage collected
// before it fires if the event loop is empty
✅ Use unref() for background tasks that shouldn't block process exit (e.g., logging, health checks).
Advanced

timeout.refresh() / timeout[Symbol.toPrimitive]

timeout.refresh() / timeout[Symbol.toPrimitive]()

Reset the timer to the original delay. Useful for extending timeouts. Symbol.toPrimitive returns the timeout ID.

const timeout = setTimeout(() => {
  console.log('This will be delayed');
}, 1000);

// Reset the timer (extends by 1000ms each time)
setInterval(() => {
  timeout.refresh(); // Keeps postponing the callback
}, 500);

// Get the numeric timeout ID
const id = timeout[Symbol.toPrimitive]();
// Equivalent: Number(timeout) or +timeout
console.log('Timeout ID:', id);
✅ Use refresh() for keep-alive mechanisms or idle timeouts (e.g., session timeouts).
🔄 setInterval & clearInterval
Interval

setInterval / clearInterval

setInterval(callback, delay?, ...args) / clearInterval(interval)

Schedule a callback repeatedly with a fixed delay between executions.

// Basic interval
let count = 0;
const interval = setInterval(() => {
  count++;
  console.log(`Tick ${count}`);
  
  if (count === 5) {
    clearInterval(interval);
    console.log('Interval cleared!');
  }
}, 1000);

// With arguments
setInterval((msg) => {
  console.log(msg);
}, 2000, 'Repeating message');
⚠️ Intervals are not guaranteed to run exactly on schedule — they may drift due to event loop congestion.
Advanced

interval.ref() / interval.unref() / hasRef()

interval.ref() / interval.unref() / interval.hasRef()

Same as timeout — control whether the interval keeps the event loop alive.

const interval = setInterval(() => {
  console.log('Background tick');
}, 1000);

// Don't block process exit
interval.unref();

// Process will exit even if interval is still active
setTimeout(() => {
  console.log('Exiting...');
  process.exit(0);
}, 3000);
✅ Use unref() for background polling or metrics collection that shouldn't prevent shutdown.
⚡ setImmediate & clearImmediate
Immediate

setImmediate / clearImmediate

setImmediate(callback, ...args) / clearImmediate(immediate)

Execute callback on the next event loop iteration (after I/O events, before timers).

// Basic immediate
const immediate = setImmediate(() => {
  console.log('Runs on the next event loop tick');
});

// With arguments
setImmediate((msg) => {
  console.log(msg);
}, 'Immediate with args');

// Clear before execution
const imm = setImmediate(() => {
  console.log('Will never run');
});
clearImmediate(imm);

// Execution order: setImmediate vs setTimeout(0)
setTimeout(() => {
  console.log('Timeout 0');
}, 0);

setImmediate(() => {
  console.log('Immediate'); // Usually runs before setTimeout(0)
});
✅ Use setImmediate for breaking up long-running synchronous operations (yield to the event loop).
Advanced

immediate.ref() / immediate.unref() / hasRef()

immediate.ref() / immediate.unref() / immediate.hasRef()

Same as timeout/interval — control whether the immediate keeps the event loop alive.

const imm = setImmediate(() => {
  console.log('Will run, but not keep process alive');
});

// Unref: process can exit before this runs
imm.unref();

// If process exits now, the immediate may not run
process.exit(0);
⚠️ If the event loop is empty and the immediate is unref'd, it may be garbage collected before execution.
🔧 Utility Functions
Util

setTimeout / setInterval / setImmediate with Promises

promisify / Promise wrappers

Wrap timers in Promises for async/await usage.

import { promisify } from 'node:util';

// Promise-based setTimeout
function sleep(ms) {
  return new Promise(resolve => setTimeout(resolve, ms));
}

// Usage in async functions
async function example() {
  console.log('Start');
  await sleep(2000);
  console.log('After 2 seconds');
  
  // setImmediate as Promise
  await new Promise(resolve => setImmediate(resolve));
  console.log('After next tick');
}

// Using promisify (for Node.js 18+)
import { setTimeout as setTimeoutPromise } from 'node:timers/promises';
await setTimeoutPromise(1000); // Promises version (Node.js 16+)
✅ Node.js 16+ has built-in Promise versions in node:timers/promises.
Util

Event Loop Ordering

Execution order: microtasks → timers → I/O → check → close

Understanding where timers run in the event loop phases.

// Microtasks (process.nextTick, Promise.then) run FIRST
process.nextTick(() => {
  console.log('1: process.nextTick');
});

Promise.resolve().then(() => {
  console.log('2: Promise.then');
});

// Timers phase
setTimeout(() => {
  console.log('3: setTimeout');
}, 0);

// Check phase (setImmediate)
setImmediate(() => {
  console.log('4: setImmediate');
});

// Output order: 1, 2, 3, 4 (usually)
// But setTimeout(0) vs setImmediate order can vary!
ℹ️ setImmediate runs in the "check" phase, after I/O events. setTimeout(0) runs in the "timers" phase.
🚀 Pro Pattern — Idle Timeout with Refresh: 
class IdleTimeout {
  constructor(timeoutMs, onTimeout) {
    this.timeoutMs = timeoutMs;
    this.onTimeout = onTimeout;
    this._timer = null;
    this.reset();
  }

  reset() {
    if (this._timer) {
      clearTimeout(this._timer);
    }
    this._timer = setTimeout(() => {
      this._timer = null;
      this.onTimeout();
    }, this.timeoutMs);
    this._timer.unref(); // Don't block process exit
  }

  clear() {
    if (this._timer) {
      clearTimeout(this._timer);
      this._timer = null;
    }
  }
}

// Usage: reset on user activity
const idle = new IdleTimeout(30000, () => {
  console.log('User inactive for 30s');
});
// On user activity: idle.reset();

⚠️ Common Pitfalls

MistakeWhy It's WrongFix
Using setTimeout(fn, 0) for "immediate" executionRuns after I/O, not immediatelyUse setImmediate() or process.nextTick()
Assuming precise timingEvent loop congestion causes delaysTreat delays as minimum, not exact
Not clearing intervalsMemory leaks, resource wasteAlways clear intervals when done
Using unref() on critical timersProcess may exit before timer firesUse ref() for critical operations
Nested setTimeout without clearingAccumulating timers, memory leaksTrack and clear nested timers
Heavy work in timersBlocks the event loopUse setImmediate to break up work
