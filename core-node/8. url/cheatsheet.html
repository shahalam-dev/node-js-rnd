🌐 Node.js url Module Cheat Sheet

URL parsing, formatting, and resolution — Node.js 18+ (WHATWG & Legacy)

💡 Golden Rule: Prefer the WHATWG URL API (new URL()) over legacy methods. It's more robust and handles all modern URL features. Import with node:url prefix.
📦 WHATWG URL API (Recommended)
Parse

new URL()

new URL(input, base?)

Create a URL object. Parses and validates. Throws on invalid URLs.

import { URL } from 'node:url';

const url = new URL('https://user:pass@example.com:8080/path?query=1#hash');

// Access properties
url.protocol    // 'https:'
url.username    // 'user'
url.password    // 'pass'
url.hostname    // 'example.com'
url.port        // '8080'
url.pathname    // '/path'
url.search      // '?query=1'
url.hash        // '#hash'
url.origin      // 'https://example.com:8080'

// Resolve relative
const base = new URL('https://example.com/dir/');
const rel = new URL('../file.js', base);
// → 'https://example.com/file.js'
✅ Use for: parsing, validation, and building URLs.
Format

url.href / url.toString()

url.href / url.toString()

Get the full URL string. href and toString() are identical.

const url = new URL('https://example.com');
url.href           // 'https://example.com/'
url.toString()      // 'https://example.com/'

// Mutate and re-serialize
url.pathname = '/api/v2';
url.search = '?key=value';
url.href           // 'https://example.com/api/v2?key=value'
ℹ️ href is the canonical string representation.
Parse

URLSearchParams

url.searchParams

Iterable interface for query parameters. Similar to Map/URLSearchParams.

const url = new URL('https://example.com?foo=1&bar=2&foo=3');

// Get first value
url.searchParams.get('foo');   // '1'

// Get all values
url.searchParams.getAll('foo'); // ['1', '3']

// Iterate
for (const [key, val] of url.searchParams) {
  console.log(key, val);
}

// Modify
url.searchParams.set('foo', '42');
url.searchParams.append('bar', 'new');
url.searchParams.delete('baz');
url.href // reflects changes
✅ Use for: reading, modifying, or building query strings.
Util

url.domainToASCII / domainToUnicode

url.domainToASCII(domain) / url.domainToUnicode(domain)

Convert between Unicode domain names (IDN) and ASCII Punycode.

import { domainToASCII, domainToUnicode } from 'node:url';

domainToASCII('español.com');    // 'xn--espaol-zwa.com'
domainToUnicode('xn--espaol-zwa.com'); // 'español.com'

// Used internally by URL class
const url = new URL('https://español.com');
url.hostname // 'xn--espaol-zwa.com' (ASCII)
📜 Legacy URL API (Node.js specific)
Parse

url.parse()

url.parse(urlString, parseQueryString?, slashesDenoteHost?)

Parse URL into an object. Returns Url object. Legacy – prefer new URL().

import url from 'node:url';

const parsed = url.parse('https://user:pass@host:8080/path?query=1#hash');
// → {
//   protocol: 'https:',
//   slashes: true,
//   auth: 'user:pass',
//   host: 'host:8080',
//   port: '8080',
//   hostname: 'host',
//   hash: '#hash',
//   search: '?query=1',
//   query: 'query=1',
//   pathname: '/path',
//   path: '/path?query=1',
//   href: 'https://user:pass@host:8080/path?query=1#hash'
// }

// Parse query into object
const withQuery = url.parse('/?foo=1&bar=2', true);
// → query: { foo: '1', bar: '2' }
⚠️ Legacy: url.parse() is not as strict as new URL(). Use new URL() for new code.
Format

url.format()

url.format(urlObject)

Build a URL string from a URL object. Works with both legacy and WHATWG objects.

const url = require('node:url');

// From legacy object
url.format({
  protocol: 'https',
  hostname: 'example.com',
  pathname: '/path',
  query: { foo: 'bar' }
});
// → 'https://example.com/path?foo=bar'

// From WHATWG URL
const whatwg = new URL('https://example.com');
url.format(whatwg); // 'https://example.com/'
ℹ️ url.format() is useful when you need to serialize a legacy URL object.
Resolve

url.resolve()

url.resolve(from, to)

Resolve a target URL relative to a base URL. Legacy – use new URL(to, from) instead.

import url from 'node:url';

url.resolve('https://example.com/dir/', '../file.js');
// → 'https://example.com/file.js'

url.resolve('https://example.com/', '/absolute');
// → 'https://example.com/absolute'

// Modern equivalent
new URL('../file.js', 'https://example.com/dir/').href;
// → 'https://example.com/file.js'
⚠️ Legacy: url.resolve() is deprecated. Use new URL() for resolution.
🔄 File URL & Utility
Util

fileURLToPath / pathToFileURL

fileURLToPath(url) / pathToFileURL(path)

Convert between file:// URLs and filesystem paths. Critical for ESM and cross-platform.

import { fileURLToPath, pathToFileURL } from 'node:url';

// File URL → filesystem path
const url = new URL('file:///C:/Users/foo/file.txt');
fileURLToPath(url); // 'C:\\Users\\foo\\file.txt' (Windows)
fileURLToPath('file:///home/user/file.txt'); // '/home/user/file.txt'

// Path → file:// URL
pathToFileURL('/home/user/file.txt').href;
// 'file:///home/user/file.txt'

// __dirname in ESM
const __filename = fileURLToPath(import.meta.url);
✅ Use for: converting between file:// URLs and filesystem paths.
Util

url.urlToHttpOptions

url.urlToHttpOptions(url)

Convert a WHATWG URL to an options object for http.request() or https.request().

import { urlToHttpOptions } from 'node:url';
import https from 'node:https';

const url = new URL('https://api.example.com/data?q=1');
const options = urlToHttpOptions(url);
// → { protocol: 'https:', hostname: 'api.example.com', ... }

https.request(options, res => { /* ... */ });
ℹ️ Useful for Node.js HTTP clients that expect legacy options objects.
🚀 Pro Pattern — Safe URL Construction: 
import { URL } from 'node:url';

// Build URL with dynamic segments (safe & auto-encoded)
function buildUrl(base, pathSegments, query) {
  const url = new URL(base);
  url.pathname = path.join(url.pathname, ...pathSegments);
  for (const [k, v] of Object.entries(query)) {
    url.searchParams.set(k, v);
  }
  return url;
}

// Use it
const apiUrl = buildUrl('https://api.com/v1', ['users', '123'], { fields: 'name,email' });
// → 'https://api.com/v1/users/123?fields=name%2Cemail'

MistakeWhy It's WrongFix
