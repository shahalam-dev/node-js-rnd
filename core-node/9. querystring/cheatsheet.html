🔢 Node.js querystring Module Cheat Sheet

Parse and stringify URL query strings — Node.js 18+

💡 Golden Rule: For new projects, consider using WHATWG URLSearchParams (built into Node.js) as a more modern alternative. The querystring module is still useful for legacy compatibility and specific edge cases. Import with node:querystring prefix.
📖 Parsing
Parse

querystring.parse

qs.parse(str, sep?, eq?, options?)

Parse a query string into an object. Handles URL encoding (e.g., %20 → space).

import qs from 'node:querystring';

// Basic parse
qs.parse('foo=bar&baz=qux&baz=quux&corge');
// → { foo: 'bar', baz: ['qux', 'quux'], corge: '' }

// Custom separator & delimiter
qs.parse('foo=bar;baz=qux', ';');
// → { foo: 'bar', baz: 'qux' }

// With custom eq sign
qs.parse('foo:bar,baz:qux', ',', ':');
// → { foo: 'bar', baz: 'qux' }

// Max keys limit (security)
qs.parse('a=1&b=2&c=3', '&', '=', { maxKeys: 2 });
// → { a: '1', b: '2' }  (ignores 'c=3')

// Decode URI components automatically
qs.parse('q=hello%20world');
// → { q: 'hello world' }
✅ Use for: parsing query strings from URLs, form data, or legacy APIs.
Parse

Handling Arrays & Objects

Array syntax & nested objects

querystring.parse does NOT support nested objects or array syntax by default. It only supports flat key-value pairs.

// 💡 querystring.parse does NOT create nested objects
qs.parse('a[b]=1&a[c]=2');
// → { 'a[b]': '1', 'a[c]': '2' }  (not nested!)

// ✅ For nested objects, use URLSearchParams or a library
const params = new URLSearchParams('a[b]=1&a[c]=2');
// But even URLSearchParams doesn't create nested objects.

// ⚠️ Many npm packages (like qs) support nested objects.
// For built-in, flatten your data before stringifying.
⚠️ querystring.parse is flat by design. For nested structures, use the qs npm package or handle manually.
✏️ Stringifying
Stringify

querystring.stringify

qs.stringify(obj, sep?, eq?, options?)

Convert an object to a query string. URL-encodes keys and values.

import qs from 'node:querystring';

// Basic stringify
qs.stringify({ foo: 'bar', baz: ['qux', 'quux'], corge: '' });
// → 'foo=bar&baz=qux&baz=quux&corge='

// Custom separators
qs.stringify({ foo: 'bar', baz: 'qux' }, ';', ':');
// → 'foo:bar;baz:qux'

// Encode URI components
qs.stringify({ q: 'hello world', special: '$&+,/:;=?@' });
// → 'q=hello%20world&special=%24%26%2B%2C%2F%3A%3B%3D%3F%40'

// encodeURIComponent options (escape)
qs.stringify({ a: 'b' }, '&', '=', {
  encodeURIComponent: str => encodeURIComponent(str).replace(/\(/g, '%28')
});
✅ Use for: building query strings for URLs, API calls, or redirects.
Stringify

Handling null & undefined

Stringifying special values

null and undefined are stringified as empty strings. Use encodeURIComponent for custom handling.

const qs = require('node:querystring');

qs.stringify({ a: null, b: undefined, c: 'value' });
// → 'a=&b=&c=value'

// Custom encode to skip null/undefined
const filtered = obj => Object.fromEntries(
  Object.entries(obj).filter(([, v]) => v != null)
);
qs.stringify(filtered({ a: null, b: 'value' }));
// → 'b=value'
⚠️ Be careful: null and undefined become empty strings, which might not be intended.
🔧 Escaping & Unescaping
Escape

querystring.escape

qs.escape(str)

Percent-encode a string for use in a query string. Similar to encodeURIComponent but with slightly different behavior.

import qs from 'node:querystring';

qs.escape('hello world');
// → 'hello%20world'

qs.escape('$&+,/:;=?@');
// → '%24%26%2B%2C%2F%3A%3B%3D%3F%40'

// Difference from encodeURIComponent:
encodeURIComponent("'"); // '%27'
qs.escape("'");       // "'" (not encoded!)

encodeURIComponent('!'); // '%21'
qs.escape('!');       // '!' (not encoded!)
⚠️ qs.escape does not encode ! and ' by default. Use with caution.
Escape

querystring.unescape

qs.unescape(str)

Decode a percent-encoded string. Similar to decodeURIComponent.

import qs from 'node:querystring';

qs.unescape('hello%20world');
// → 'hello world'

qs.unescape('%24%26%2B%2C%2F%3A%3B%3D%3F%40');
// → '$&+,/:;=?@'

// Difference: qs.unescape uses the decode function from the module
decodeURIComponent('%27'); // "'"
qs.unescape('%27');      // "'" (same)
ℹ️ unescape is used internally by parse to decode values.
🧪 Comparison: querystring vs URLSearchParams
Util

querystring vs URLSearchParams

When to use which?

Both parse & stringify query strings, but with different capabilities.

// querystring (Node.js built-in)
import qs from 'node:querystring';
qs.parse('a=1&a=2');  // { a: ['1', '2'] }
qs.stringify({ a: ['1', '2'] }); // 'a=1&a=2'

// URLSearchParams (WHATWG standard)
const sp = new URLSearchParams('a=1&a=2');
sp.get('a');  // '1' (first value)
sp.getAll('a'); // ['1', '2']
sp.toString(); // 'a=1&a=2'

// 🔥 URLSearchParams supports iterative methods
for (const [k, v] of sp) { /* ... */ }
sp.has('a');  // true
sp.set('a', '3');
sp.delete('a');
✅ Recommendation: Use URLSearchParams for new code. It's standard and works in browsers too.
🚀 Pro Pattern — Safe Query String Building: 
import qs from 'node:querystring';

// Build query string safely with filtering
function buildQuery(params) {
  // Remove null/undefined values
  const clean = Object.fromEntries(
    Object.entries(params).filter(([, v]) => v != null)
  );
  return qs.stringify(clean);
}

// Use with URL
const url = `https://api.com?${buildQuery({ page: 1, limit: 20, filter: null })}`;
// → 'https://api.com?page=1&limit=20'

// For arrays, use duplicate keys
qs.stringify({ id: ['1', '2', '3'] });
// → 'id=1&id=2&id=3'
