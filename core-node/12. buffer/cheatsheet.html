📦 Node.js Buffer Module Cheat Sheet

Binary data handling — Node.js 18+ (Global API, no import needed)

💡 Golden Rule: Buffer is a global class — no import required. Always use Buffer.from(), Buffer.alloc(), or Buffer.allocUnsafe() to create buffers. Avoid new Buffer() (deprecated). Buffers are fixed-size and mutable.
🏗️ Creating Buffers
Create

Buffer.from()

Buffer.from(array) / Buffer.from(string, encoding?) / Buffer.from(buffer)

Create a buffer from various sources. Most common and safest way.

// From string (default utf-8)
const buf1 = Buffer.from('hello');
// → 

// From string with encoding
const buf2 = Buffer.from('68656c6c6f', 'hex');
// → 

// From array of bytes
const buf3 = Buffer.from([0x68, 0x65, 0x6c, 0x6c, 0x6f]);

// From another buffer (copy)
const buf4 = Buffer.from(buf1);
✅ Use Buffer.from() for converting strings or arrays to buffers.
Create

Buffer.alloc() / allocUnsafe()

Buffer.alloc(size, fill?, encoding?) / Buffer.allocUnsafe(size)

Allocate a buffer of a specific size. alloc initializes with zeros (safe). allocUnsafe is faster but may contain old data.

// Allocate 10 bytes initialized to 0
const buf1 = Buffer.alloc(10);
// → 

// Allocate and fill with a value
const buf2 = Buffer.alloc(5, 'a');
// → 

// Unsafe allocation (faster but may contain garbage)
const buf3 = Buffer.allocUnsafe(1024);
// ⚠️ Contains random old data — must fill before use
buf3.fill(0); // Clear it
✅ Use Buffer.alloc() for safety. Use Buffer.allocUnsafe() only when performance is critical and you'll fill it immediately.
Create

Buffer.allocUnsafeSlow() / Buffer.byteLength

Buffer.allocUnsafeSlow(size) / Buffer.byteLength(string, encoding?)

Allocate large buffers without pooling. Get byte length of a string without creating a buffer.

// For large buffers (> 4KB) to avoid shared memory pool
const large = Buffer.allocUnsafeSlow(8192);

// Get byte length (not character length!)
const len = Buffer.byteLength('hello');      // 5
const len2 = Buffer.byteLength('😊', 'utf-8'); // 4 (emoji is 4 bytes)
ℹ️ Use byteLength to pre-allocate buffers of the right size for strings.
📖 Reading Buffers
Read

toString() & encodings

buf.toString(encoding?, start?, end?)

Convert buffer to string. Supports various encodings: 'utf-8', 'hex', 'base64', 'latin1', and more.

const buf = Buffer.from('hello world');

buf.toString();                // 'hello world' (default utf-8)
buf.toString('hex');           // '68656c6c6f20776f726c64'
buf.toString('base64');        // 'aGVsbG8gd29ybGQ='
buf.toString('latin1');        // 'hello world'

// Substring (byte indices)
buf.toString('utf-8', 0, 5); // 'hello'
✅ Common encodings: 'utf8', 'hex', 'base64', 'latin1' (binary).
Read

toJSON() & .values()

buf.toJSON() / buf.values() / buf.entries()

Convert buffer to JSON or iterate over bytes.

const buf = Buffer.from([1, 2, 3, 4]);

// JSON representation
buf.toJSON();
// → { type: 'Buffer', data: [1, 2, 3, 4] }

// Iterate over bytes (values)
for (const byte of buf.values()) {
  console.log(byte);
}

// Iterate with indices
for (const [index, byte] of buf.entries()) {
  console.log(index, byte);
}
✏️ Writing & Modifying
Write

buf.write() & .fill()

buf.write(string, offset?, length?, encoding?) / buf.fill(value, offset?, end?)

Write data to a buffer at a specific position. Fill the buffer with a value.

const buf = Buffer.alloc(10);

// Write string at offset 0
buf.write('hello');
// → 

// Write at offset 5
buf.write('world', 5);
// → 

// Fill with a value
const buf2 = Buffer.alloc(5);
buf2.fill(0x61); // fill with 'a' (97)
// → 

// Fill a range
buf2.fill(0x62, 1, 3); // fill positions 1-2 with 'b'
⚠️ write() returns the number of bytes written. Be careful not to exceed buffer length.
Write

buf.set() & .copy()

buf.set(array, offset?) / buf.copy(target, targetStart?, sourceStart?, sourceEnd?)

Copy data between buffers or from arrays.

const src = Buffer.from('hello');
const dst = Buffer.alloc(10);

// Copy src to dst starting at offset 2
src.copy(dst, 2);
// → dst: 

// Copy only part of src
src.copy(dst, 0, 0, 3); // copy first 3 bytes ('hel')

// Set bytes from array
const buf = Buffer.alloc(4);
buf.set([1, 2, 3, 4]);
✅ copy() is efficient for moving data between buffers.
🔧 Buffer Utilities
Util

buf.slice() & .subarray()

buf.slice(start?, end?) / buf.subarray(start?, end?)

Create a new buffer that references the same memory (no copy). slice and subarray are similar but with different behavior.

const buf = Buffer.from('hello world');

// Slice (views original memory)
const slice = buf.slice(0, 5);
// →  (references same memory)

// Modifying slice affects original
slice[0] = 0x48; // 'H'
// buf now: 'Hello world'

// subarray is similar but more aligned with TypedArray
const sub = buf.subarray(6); // 'world'
⚠️ slice() and subarray() do not copy data — they share memory. Use Buffer.from() to create a copy.
Util

Buffer.concat() & .compare()

Buffer.concat(list, totalLength?) / buf.compare(target)

Concatenate multiple buffers. Compare buffers for sorting or equality.

const buf1 = Buffer.from('hello ');
const buf2 = Buffer.from('world');

// Concatenate
const combined = Buffer.concat([buf1, buf2]);
// → 

// Compare buffers
const a = Buffer.from('abc');
const b = Buffer.from('def');
a.compare(b); // -1 (a < b)
b.compare(a); // 1 (b > a)
a.compare(a); // 0 (equal)
✅ Use concat for joining buffers. Use compare for sorting or equality checks.
Util

buf.indexOf() & .includes()

buf.indexOf(value, byteOffset?, encoding?) / buf.includes(value, byteOffset?, encoding?)

Search for a value in the buffer. Returns index or -1.

const buf = Buffer.from('hello world');

buf.indexOf('world');   // 6
buf.indexOf('o');        // 4
buf.indexOf('o', 5);    // 7 (search from offset 5)
buf.indexOf(0x68);      // 0 ('h')

buf.includes('hello');   // true
buf.includes('xyz');      // false
Util

buf.isBuffer() & .isEncoding()

Buffer.isBuffer(obj) / Buffer.isEncoding(encoding)

Check if an object is a buffer. Check if an encoding is supported.

const buf = Buffer.from('hello');
const str = 'hello';

Buffer.isBuffer(buf);  // true
Buffer.isBuffer(str);  // false

Buffer.isEncoding('utf-8');   // true
Buffer.isEncoding('hex');      // true
Buffer.isEncoding('invalid'); // false
✅ Use isBuffer to validate buffer inputs. Use isEncoding to check encoding support.
🚀 Pro Pattern — Efficient Buffer Operations: 
// Safe string to buffer with proper encoding
function safeBuffer(str, encoding = 'utf-8') {
  if (!Buffer.isEncoding(encoding)) {
    throw new Error(`Invalid encoding: ${encoding}`);
  }
  return Buffer.from(str, encoding);
}

// Chunked buffer concatenation with pre-allocation
function concatBuffers(buffers) {
  const totalLen = buffers.reduce((sum, b) => sum + b.length, 0);
  const result = Buffer.allocUnsafe(totalLen);
  let offset = 0;
  for (const buf of buffers) {
    buf.copy(result, offset);
    offset += buf.length;
  }
  return result;
}

// Read integer from buffer (big endian)
const buf = Buffer.from([0x00, 0x00, 0x01, 0x2A]);
const num = buf.readUInt32BE(0); // 298

⚠️ Common Pitfalls

MistakeWhy It's WrongFix
new Buffer(size)Deprecated, behaves unpredictablyUse Buffer.alloc() or Buffer.from()
Mutating slice() outputModifies original buffer tooUse Buffer.from(slice) for a copy
Assuming toString() returns full stringTruncation if buffer contains partial multi-byte charsUse string_decoder for multi-byte safety
Using allocUnsafe() without fillingContains old sensitive dataAlways fill with .fill(0) or overwrite
Not checking buffer length before writeBuffer overflow or truncationCheck buf.length and write length
Using == to compare buffersCompares references, not contentUse buf.compare() or buf.equals()
