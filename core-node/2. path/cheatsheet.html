🔗 Node.js path Module Cheat Sheet

Cross-platform path manipulation — Node.js 18+

💡 Golden Rule: Always use path.join() or path.resolve() instead of string concatenation. Never hardcode '/' or '\' separators. Import with node:path prefix.
🏗️ Building Paths
Build

path.join

path.join(...paths)

Join path segments using the platform-specific separator. Normalizes the result.

import path from 'node:path';

path.join('/foo', 'bar', 'baz/asdf', 'quux', '..');
// → '/foo/bar/baz/asdf'

path.join('foo', 'bar');
// → 'foo/bar' (Linux/Mac) or 'foo\bar' (Windows)

// Safe: handles extra slashes automatically
path.join('src/', '/components', 'Button.js');
// → 'src/components/Button.js'
✅ Use for: combining relative path segments, building file paths.
Build

path.resolve

path.resolve(...paths)

Resolve to an absolute path. Starts from current working directory and navigates through segments. Think of it as a series of cd commands.

path.resolve('foo/bar');
// → '/current/working/dir/foo/bar'

path.resolve('/foo/bar', './baz');
// → '/foo/bar/baz'

path.resolve('/foo/bar', '/tmp/file/');
// → '/tmp/file' (absolute overrides previous)

path.resolve('wwwroot', 'static_files/png/', '../gif/image.gif');
// → '/cwd/wwwroot/static_files/gif/image.gif'
✅ Use for: converting relative paths to absolute, resolving config paths.
Build

path.normalize

path.normalize(path)

Clean up a path by resolving .. and . segments, removing duplicate slashes, and trailing slashes.

path.normalize('/foo/bar//baz/asdf/quux/..');
// → '/foo/bar/baz/asdf'

path.normalize('foo\bar\baz');
// → 'foo/bar/baz' (on POSIX) or 'foo\bar\baz' (on Windows)

path.normalize('C:\\temp\\\foo\bar');
// → 'C:\temp\foo\bar' (Windows)
ℹ️ Usually not needed explicitly — join and resolve normalize automatically.
🔍 Parsing Paths
Parse

path.basename

path.basename(path, ext?)

Get the last portion (filename) of a path. Optionally strip an extension.

path.basename('/foo/bar/baz.html');
// → 'baz.html'

path.basename('/foo/bar/baz.html', '.html');
// → 'baz'

path.basename('C:\\foo\bar\baz.txt');
// → 'baz.txt' (Windows)
⚠️ Ignores trailing separators: basename('/foo/bar/') → 'bar'
Parse

path.dirname

path.dirname(path)

Get the directory portion of a path.

path.dirname('/foo/bar/baz.txt');
// → '/foo/bar'

path.dirname('/foo/bar/');
// → '/foo'

path.dirname('baz.txt');
// → '.' (current directory)
Parse

path.extname

path.extname(path)

Get the file extension from the last . in the basename.

path.extname('index.html');
// → '.html'

path.extname('app.min.js');
// → '.js'

path.extname('README');
// → '' (no extension)

path.extname('.gitignore');
// → '' (hidden files have no extension)
Parse

path.parse / path.format

path.parse(path)

Break a path into an object with root, dir, base, ext, name. format does the reverse.

path.parse('/home/user/dir/file.txt');
// → {
//   root: '/',
//   dir: '/home/user/dir',
//   base: 'file.txt',
//   ext: '.txt',
//   name: 'file'
// }

// Reconstruct
path.format({
  dir: '/home/user',
  base: 'file.txt'
});
// → '/home/user/file.txt'
✅ Checks & Comparison
Check

path.isAbsolute

path.isAbsolute(path)

Check if a path is absolute (platform-aware).

path.isAbsolute('/foo/bar');      // → true (POSIX)
path.isAbsolute('C:\\foo');     // → true (Windows)
path.isAbsolute('foo/bar');      // → false
path.isAbsolute('..');           // → false
💡 Use before path.resolve() to avoid unnecessary absolute conversion.
Check

path.relative

path.relative(from, to)

Get the relative path from from to to. Think of it as: "what path do I need to get from A to B?"

path.relative('/data/orandea/test/aaa', '/data/orandea/impl/bbb');
// → '../../impl/bbb'

path.relative('/data/orandea/test/aaa', '/data/orandea/test/aaa/xxx');
// → 'xxx'

path.relative('src/components', 'src/utils');
// → '../utils'
✅ Use for: generating relative imports, creating portable paths in logs.
🛠️ Utilities & Platform
Util

path.sep & path.delimiter

path.sep / path.delimiter

Platform-specific path separator and environment PATH delimiter.

path.sep;
// → '/' (POSIX) or '\' (Windows)

path.delimiter;
// → ':' (POSIX) or ';' (Windows)

// Practical: split PATH env variable
const paths = process.env.PATH.split(path.delimiter);
Util

path.posix / path.win32

path.posix / path.win32

Explicitly use POSIX or Windows path logic regardless of the current platform.

// Force POSIX paths (always forward slashes)
path.posix.join('foo', 'bar');
// → 'foo/bar' (even on Windows)

path.posix.normalize('C:\\foo\bar');
// → 'C:/foo/bar'

// Force Windows paths (always backslashes)
path.win32.join('foo', 'bar');
// → 'foo\bar' (even on Linux/Mac)

// Common use: generating URLs or config files
path.posix.relative('/src', '/src/components/Button.js');
// → 'components/Button.js' (portable URL path)
🌐 Use path.posix when generating paths for URLs, web configs, or tar archives.
Util

path.toNamespacedPath

path.toNamespacedPath(path)

On Windows, returns a \?\ prefixed path for extended-length paths. On POSIX, returns as-is.

path.toNamespacedPath('C:\\foo\bar');
// → '\\?\C:\foo\bar' (Windows)

path.toNamespacedPath('/foo/bar');
// → '/foo/bar' (POSIX, unchanged)
ℹ️ Rarely needed directly. Used internally by fs module on Windows.
🚀 Pro Pattern — Safe File Path Builder: 
import path from 'node:path';
import { fileURLToPath } from 'node:url';

// __dirname in ESM modules
const __filename = fileURLToPath(import.meta.url);
const __dirname = path.dirname(__filename);

// Build absolute paths relative to project root
const configPath = path.resolve(__dirname, '../config/app.json');
const uploadDir  = path.join(__dirname, 'uploads');

// Change extension
const { dir, name } = path.parse('./src/app.js');
const tsPath = path.format({ dir, name, ext: '.ts' });
// → './src/app.ts'
