🛠️ Node.js util Module Cheat Sheet

Utility functions for everyday Node.js tasks — Node.js 18+

💡 Golden Rule: The util module bridges callback-style APIs with modern Promises, provides deep inspection for debugging, and offers handy formatting utilities. Most of it is for development and interoperability — not production hot paths.
⏳ Async Utilities
Async

util.promisify

util.promisify(original)

Convert a callback-based function into a Promise-returning function. The most important util function for modernizing legacy code.

import { promisify } from 'node:util';
import fs from 'node:fs';

// Convert fs.readFile (callback) to Promise
const readFileAsync = promisify(fs.readFile);

const data = await readFileAsync('./file.txt', 'utf-8');

// Works with any callback-last, (err, result) function
const execAsync = promisify(require('node:child_process').exec);
const { stdout } = await execAsync('ls -la');
✅ Most core modules now have /promises exports (e.g., node:fs/promises). Use those instead of promisify when available.
Async

util.callbackify

util.callbackify(original)

The reverse of promisify. Convert an async/Promise function into a callback-style function.

import { callbackify } from 'node:util';

async function fetchData(url) {
  const res = await fetch(url);
  return await res.json();
}

// Convert to callback style for legacy compatibility
const fetchDataCb = callbackify(fetchData);

fetchDataCb('https://api.example.com', (err, data) => {
  if (err) console.error(err);
  else console.log(data);
});
ℹ️ Rarely needed today. Useful when integrating with older callback-based libraries.
Async

util.parseArgs

util.parseArgs(config)

Parse command-line arguments. Built-in replacement for minimist and yargs-parser for simple cases.

import { parseArgs } from 'node:util';

const { values } = parseArgs({
  options: {
    port: { type: 'string', short: 'p', default: '3000' },
    verbose: { type: 'boolean', short: 'v' },
    env: { type: 'string', multiple: true }
  },
  allowPositionals: true
});

// node app.js --port 8080 -v production
values.port      // '8080'
values.verbose   // true
values.env       // ['production']
✅ Added in Node.js 18.3. No external dependency needed for CLI argument parsing.
🔍 Debugging & Inspection
Debug

util.inspect

util.inspect(object, options?)

Deep, human-readable string representation of any JavaScript value. The engine behind console.log for objects.

import { inspect } from 'node:util';

const obj = { a: 1, b: { c: ['x', 'y'] }, fn: () => {} };

inspect(obj);
// → "{ a: 1, b: { c: [ 'x', 'y' ] }, fn: [Function: fn] }"

// With options
inspect(obj, {
  depth: null,        // unlimited depth
  colors: true,       // ANSI colors
  showHidden: true,   // non-enumerable props
  maxArrayLength: 10,
  breakLength: 80
});

// Custom inspect via Symbol
class User {
  [Symbol.for('nodejs.util.inspect.custom')](depth, options) {
    return `User: ${this.name}`;
  }
}
Debug

util.format

util.format(format, ...args)

String formatting with printf-style placeholders. The engine behind console.log formatting.

import { format } from 'node:util';

format('Hello %s, you have %d messages', 'Alice', 5);
// → 'Hello Alice, you have 5 messages'

format('%o', { a: 1 });      // Object (inspect)
format('%O', { a: 1 });      // Object (no colors)
format('%j', { a: 1 });      // JSON string
format('%d', 42);            // Number (integer)
format('%f', 3.14);           // Number (float)
format('%s', 'text');        // String
format('%%');                 // Literal '%'

// Extra args are inspect()-ed and concatenated
format('User:', { name: 'Bob' });
// → "User: { name: 'Bob' }"
ℹ️ console.log uses util.format internally. You rarely need to call it directly.
Debug

util.debuglog / util.debug

util.debuglog(section)

Conditional debug logging controlled by the NODE_DEBUG environment variable.

import { debuglog } from 'node:util';

const debug = debuglog('http');

debug('Processing request %o', { url: '/api' });

// Only prints when: NODE_DEBUG=http node app.js
// Output: HTTP 3245: Processing request { url: '/api' }

// Multiple sections
const dbDebug = debuglog('db');
// NODE_DEBUG=http,db  (comma-separated)
✅ Use for library debug output. Users opt-in via NODE_DEBUG=yoursection.
📋 Type Checking
Type

util.types — Fast Native Type Checks

Reliable type checking that works across realms and frames, unlike instanceof.

import { types } from 'node:util';

types.isPromise(new Promise(() => {}));     // true
types.isAsyncFunction(async () => {});      // true
types.isDate(new Date());                   // true
types.isRegExp(/abc/);                         // true
types.isMap(new Map());                       // true
types.isSet(new Set());                       // true
types.isArrayBuffer(new ArrayBuffer(8));     // true
types.isSharedArrayBuffer(sab);               // true
types.isTypedArray(new Uint8Array(4));      // true
types.isDataView(new DataView(buffer));      // true
types.isProxy(new Proxy({}, {}));             // true
types.isWeakMap(new WeakMap());               // true
types.isWeakSet(new WeakSet());               // true
types.isGeneratorObject((function*() {})()); // true
types.isModuleNamespaceObject(ns);            // true (import * as ns)

// Why not instanceof? Fails across iframes/VM contexts
obj instanceof Date   // false in different realm
types.isDate(obj)     // true always
✅ Prefer util.types over instanceof when checking built-in types. It's faster and works across realms.
⚠️ Deprecation Utilities
Deprecate

util.deprecate

util.deprecate(fn, msg, code?)

Wrap a function to emit a deprecation warning on first call. Used by Node.js core for API evolution.

import { deprecate } from 'node:util';

const oldApi = deprecate(() => {
  // old behavior
}, 'oldApi() is deprecated. Use newApi() instead.', 'DEP0001');

oldApi();
// (node:1234) [DEP0001] DeprecationWarning:
// oldApi() is deprecated. Use newApi() instead.
Deprecate

util.toUSVString

util.toUSVString(string)

Convert a string to a well-formed Unicode string by replacing lone surrogates.

import { toUSVString } from 'node:util';

toUSVString('\uD800');  // '\uFFFD' (replacement character)
toUSVString('hello');     // 'hello' (unchanged)
ℹ️ Rarely needed directly. Used internally for URL and Web API compliance.
🔧 Miscellaneous
Misc

util.inherits

util.inherits(constructor, superConstructor)

Prototype inheritance setup. Deprecated — use class extends instead.

// OLD WAY (don't use)
function MyError(message) {
  Error.call(this, message);
}
inherits(MyError, Error);

// MODERN WAY (use this)
class MyError extends Error {
  constructor(message) {
    super(message);
    this.name = 'MyError';
  }
}
⚠️ util.inherits is deprecated. Use native class extends syntax.
Misc

util.TextEncoder / util.TextDecoder

Encode/decode between strings and TypedArrays. Also available as global Web API objects.

import { TextEncoder, TextDecoder } from 'node:util';
// Or use global: new TextEncoder()

const encoder = new TextEncoder();
const decoder = new TextDecoder('utf-8');

// String → Uint8Array
const bytes = encoder.encode('Hello');
// → Uint8Array [72, 101, 108, 108, 111]

// Uint8Array → String
const str = decoder.decode(bytes);
// → 'Hello'

// Decode stream chunks
const chunk = new Uint8Array([0xF0, 0x9F, 0x98, 0x80]);
decoder.decode(chunk);  // '😀'
Misc

util.transferableAbortSignal / AbortController

Utilities for working with AbortController across threads and contexts.

import { transferableAbortSignal, transferableAbortController } from 'node:util';

// Create an AbortController whose signal can be posted to Worker threads
const controller = transferableAbortController();
const signal = transferableAbortSignal(controller.signal);

worker.postMessage({ signal }, [signal]);

// Cancel from main thread
controller.abort('User cancelled');
ℹ️ Advanced use case for Worker threads and cross-context cancellation.
🚀 Pro Pattern — Wrapping a Legacy Callback API: 
import { promisify } from 'node:util';
import { createReadStream } from 'node:fs';
import crypto from 'node:crypto';

// Wrap legacy crypto.randomBytes
const randomBytesAsync = promisify(crypto.randomBytes);

async function generateToken() {
  const buf = await randomBytesAsync(32);
  return buf.toString('hex');
}

// Custom promisify with custom behavior
const customRead = promisify((path, cb) => {
  const stream = createReadStream(path, 'utf-8');
  let data = '';
  stream.on('data', chunk => data += chunk);
  stream.on('end', () => cb(null, data));
  stream.on('error', cb);
});
