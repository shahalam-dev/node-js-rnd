🌊 Node.js stream Module Cheat Sheet

Efficient data handling with streams — Node.js 18+

💡 Golden Rule: Streams are the secret to Node.js performance. They process data chunk-by-chunk instead of loading everything into memory. Every core I/O module (fs, http, net, zlib, crypto) is built on streams.
📊 The Four Stream Types
Type Direction Example Use Base Class
Readable Source → You Reading files, HTTP requests, stdin stream.Readable
Writable You → Destination Writing files, HTTP responses, stdout stream.Writable
Duplex Both directions TCP sockets, TLS connections stream.Duplex
Transform Modify in transit Compression, encryption, CSV parsing stream.Transform
📖 Readable Streams
Readable

Consuming Readable Streams

Three ways to consume data: flowing mode, paused mode, and async iteration.

import fs from 'node:fs';

const readable = fs.createReadStream('./big-file.txt', 'utf-8');

// Method 1: Flowing mode (event-driven)
readable.on('data', (chunk) => {
  console.log('Chunk:', chunk.length);
});
readable.on('end', () => console.log('Done!'));
readable.on('error', (err) => console.error(err));

// Method 2: Paused mode (explicit read)
readable.on('readable', () => {
  let chunk;
  while ((chunk = readable.read()) !== null) {
    console.log(chunk);
  }
});

// Method 3: Async iteration (modern, cleanest)
for await (const chunk of readable) {
  console.log(chunk);
}
✅ Prefer async iteration (Method 3) for clean, sequential processing with automatic error handling.
Readable

Controlling Flow

readable.pause() / .resume() / .pipe()

Manage backpressure manually when you need fine-grained control.

// Pause the stream (stop emitting 'data')
readable.pause();

// Resume the stream
readable.resume();

// Check if in flowing mode
readable.readableFlowing;  // true | false | null

// Check if stream is readable
readable.readable;       // true until 'end'

// Destroy and cleanup
readable.destroy();        // forcibly end
readable.destroy(new Error('Aborted'));
✏️ Writable Streams
Writable

Writing to Streams

Write data chunk-by-chunk. Handle backpressure when the buffer is full.

import fs from 'node:fs';

const writable = fs.createWriteStream('./output.txt');

// Write returns boolean: true = buffer not full, false = backpressure
const ok = writable.write('Hello ');
writable.write('World!\n');

// When backpressure hits, wait for 'drain' before writing more
if (!ok) {
  writable.once('drain', () => {
    writable.write('More data...');
  });
}

// Finish writing and close the stream
writable.end('Final chunk');

writable.on('finish', () => console.log('All data written'));
writable.on('error', (err) => console.error(err));
⚠️ Always handle the 'drain' event when write() returns false to avoid memory issues with high-volume writes.
Writable

Writable Events & Properties

writable.on('finish', () => {});    // All data flushed, stream ended
writable.on('drain', () => {});     // Buffer emptied, safe to write more
writable.on('error', (err) => {});  // Write error occurred
writable.on('close', () => {});    // Stream fully closed

writable.writable;                  // true if stream is open for writing
writable.writableEnded;             // true after .end() called
writable.writableFinished;          // true after 'finish' event
writable.writableLength;            // bytes in internal buffer

writable.destroy();                 // forcibly close
🔀 Piping & Pipeline
Pipe

stream.pipe (Classic)

readable.pipe(destination, options?)

The classic way to connect streams. Simple but requires manual error handling.

import fs from 'node:fs';
import zlib from 'node:zlib';

const source = fs.createReadStream('./input.txt');
const gzip = zlib.createGzip();
const dest = fs.createWriteStream('./input.txt.gz');

// Chain pipes
source.pipe(gzip).pipe(dest);

// Manual error handling (pipe does NOT forward errors)
source.on('error', handleError);
gzip.on('error', handleError);
dest.on('error', handleError);

// Unpipe when done
dest.on('finish', () => {
  source.unpipe(gzip);
});
⚠️ pipe() does NOT handle errors automatically. A broken stream in the middle silently hangs.
Pipe

stream.pipeline (Modern)

pipeline(source, ...transforms, destination, callback)

The modern, safer way to pipe streams. Forwards errors, cleans up, and handles backpressure properly.

import { pipeline } from 'node:stream';
import fs from 'node:fs';
import zlib from 'node:zlib';

// Callback style
pipeline(
  fs.createReadStream('./input.txt'),
  zlib.createGzip(),
  fs.createWriteStream('./input.txt.gz'),
  (err) => {
    if (err) console.error('Pipeline failed:', err);
    else console.log('Pipeline succeeded!');
  }
);

// Promise style (preferred)
import { pipeline } from 'node:stream/promises';

try {
  await pipeline(
    fs.createReadStream('./input.txt'),
    zlib.createGzip(),
    fs.createWriteStream('./input.txt.gz')
  );
  console.log('Done!');
} catch (err) {
  console.error(err);
}
✅ Always use pipeline() over .pipe() in production. It handles errors, cleanup, and backpressure automatically.
🔧 Transform Streams
Transform

Creating Custom Transforms

new Transform({ transform(chunk, encoding, callback) })

Transform streams sit in the middle of a pipeline, modifying data as it flows through.

import { Transform } from 'node:stream';

// Uppercase transform
const uppercase = new Transform({
  transform(chunk, encoding, callback) {
    this.push(chunk.toString().toUpperCase());
    callback();
  }
});

// JSON parse transform
const parseJSON = new Transform({
  transform(chunk, enc, cb) {
    try {
      const obj = JSON.parse(chunk);
      this.push(JSON.stringify({ ...obj, parsed: true }));
      cb();
    } catch (err) {
      cb(err);  // Pass error to pipeline
    }
  }
});
Transform

Object Mode Streams

By default streams pass Buffers/strings. Enable objectMode to pass JavaScript objects.

import { Transform } from 'node:stream';

const doubler = new Transform({
  objectMode: true,
  transform(obj, enc, cb) {
    this.push({ ...obj, value: obj.value * 2 });
    cb();
  }
});

doubler.write({ value: 5 });   // Pass objects directly
doubler.on('data', (obj) => {
  console.log(obj);  // { value: 10 }
});
ℹ️ Object mode is great for data pipelines (ETL, log processing) but disables some byte-level optimizations.
🛠️ Utility Functions
Util

stream.Readable.from

Readable.from(iterable, options?)

Convert arrays, generators, or async iterables into a readable stream.

import { Readable } from 'node:stream';

// From array
const stream1 = Readable.from(['a', 'b', 'c']);

// From generator
function* generate() {
  yield 'chunk 1';
  yield 'chunk 2';
}
const stream2 = Readable.from(generate());

// From async generator
async function* asyncGenerate() {
  yield await fetchData(1);
  yield await fetchData(2);
}
const stream3 = Readable.from(asyncGenerate());
Util

stream.finished

finished(stream, callback) / finished(stream, options)

Wait for a stream to finish or close. Useful for cleanup and testing.

import { finished } from 'node:stream';
import { finished as finishedPromise } from 'node:stream/promises';

// Callback style
finished(readable, (err) => {
  console.log(err ? 'Failed' : 'Done');
});

// Promise style
await finishedPromise(writable);
Util

stream.compose

compose(...streams)

Compose multiple streams into a single stream. New in Node.js 20+.

import { compose } from 'node:stream';

const composed = compose(
  fs.createReadStream('input.txt'),
  zlib.createGzip(),
  fs.createWriteStream('output.gz')
);

// composed is a single stream with composed behavior
⚠️ Memory Leak Warning: Never ignore backpressure. If you write faster than the consumer can handle, the internal buffer grows unbounded and crashes with OOM. Always respect write() return value and the 'drain' event, or use pipeline() which handles this automatically.
🚀 Pro Pattern — Full Pipeline Example: 
import { pipeline } from 'node:stream/promises';
import { Transform } from 'node:stream';
import fs from 'node:fs';
import zlib from 'node:zlib';

// Custom transform: add line numbers
const lineNumberer = new Transform({
  transform(chunk, enc, cb) {
    this.lineNum = (this.lineNum || 0) + 1;
    this.push(`${this.lineNum}: ${chunk}`);
    cb();
  }
});

await pipeline(
  fs.createReadStream('./input.txt', 'utf-8'),
  lineNumberer,
  zlib.createGzip(),
  fs.createWriteStream('./output.txt.gz')
);

console.log('Pipeline complete!');

MistakeWhy It's WrongFix
Using .pipe() without error handlingSilent hangs on errorUse pipeline() instead

Not handling 'error' on streamsUnhandled error crashes processAlways attach .on('error', ...)
Buffering entire stream into memoryDefeats the purpose of streamsProcess chunks, use pipeline
